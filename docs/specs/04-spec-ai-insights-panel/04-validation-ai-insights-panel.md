## 3. Validation Issues

| Severity | Issue | Impact | Recommendation |
|---|---|---|---|
| **MEDIUM** | **No retry on Claude API failure** — `InsightsService.java:65` uses `onErrorMap` which converts errors to 503 immediately. Spec Unit 1 states: "return HTTP 503 … if the Claude API call fails **after one retry**." | Transient Claude API errors (timeouts, 529 overload) will surface to users instead of being retried once. | Add `.retry(1)` before `.onErrorMap(...)` in `InsightsService.streamInsights`: `Flux<String> stream = chatClient.prompt()...stream().content().retry(1).onErrorMap(...)` |
| **LOW** | **Analyzing placeholder lacks pulse animation** — `InsightsPanel.tsx:73` renders plain italic text. Spec Design Considerations state: "pulsing 'Analyzing your metrics...' placeholder uses the same skeleton animation as the metric cards." | Minor UX degradation; placeholder is visible and functionally correct but lacks the visual polish specified. | Apply pulse animation to the placeholder `<p>` via inline style or a CSS class: `style={{ animation: 'pulse 1.5s ease-in-out infinite', opacity: 0.6 }}` (matching `SkeletonCard.tsx`'s keyframe). |
| **LOW** | **Missing `Cache-Control: no-store` header** — `InsightsController.java` does not set `Cache-Control: no-store`. Spec Technical Considerations: "The `/api/insights` endpoint should set `Cache-Control: no-store` since each response is generated fresh." | Responses could theoretically be cached by intermediary proxies, causing stale AI insights. In practice the SSE `text/event-stream` content type prevents most caches from caching it. | Add to `InsightsController.getInsights`: `HttpServletResponse response` parameter, then `response.setHeader("Cache-Control", "no-store");` before returning the emitter. |
