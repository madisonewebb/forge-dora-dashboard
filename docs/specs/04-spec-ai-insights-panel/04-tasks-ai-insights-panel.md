# 04-tasks-ai-insights-panel

## Relevant Files

### Backend — Production Code
- `pom.xml` — Add `spring-ai-bom` to `<dependencyManagement>` and `spring-ai-anthropic-spring-boot-starter` to `<dependencies>`
- `src/main/resources/application.properties` — Add Spring AI Anthropic config: `spring.ai.anthropic.api-key` and model name
- `src/main/java/com/liatrio/dora/insights/TrendDirection.java` — Enum: `IMPROVING`, `STABLE`, `DECLINING`
- `src/main/java/com/liatrio/dora/insights/TrendDirectionCalculator.java` — Pure `@Component` that computes trend direction from a `List<WeekDataPoint>` by comparing first-half vs second-half averages
- `src/main/java/com/liatrio/dora/insights/PromptBuilder.java` — Pure `@Component` that constructs the system prompt and user prompt from a `MetricsResponse` and four `TrendDirection` values
- `src/main/java/com/liatrio/dora/insights/BuiltPrompt.java` — Java record: `record BuiltPrompt(String systemPrompt, String userPrompt)`
- `src/main/java/com/liatrio/dora/insights/InsightsUnavailableException.java` — Runtime exception thrown when `ANTHROPIC_API_KEY` is missing or the Claude API call fails after one retry
- `src/main/java/com/liatrio/dora/service/InsightsService.java` — `@Service` that checks API key availability, calls `MetricsService`, runs `TrendDirectionCalculator` for each metric, calls `PromptBuilder`, and streams tokens via Spring AI `ChatClient`; returns `Flux<String>`
- `src/main/java/com/liatrio/dora/controller/InsightsController.java` — `@RestController` exposing `GET /api/insights` as `text/event-stream` using `SseEmitter`
- `src/main/java/com/liatrio/dora/exception/GlobalExceptionHandler.java` — Add handler for `InsightsUnavailableException` → HTTP 503

### Backend — Test Code
- `src/test/java/com/liatrio/dora/insights/TrendDirectionCalculatorTest.java` — Plain JUnit 5; tests all trend direction scenarios and edge cases
- `src/test/java/com/liatrio/dora/insights/PromptBuilderTest.java` — Plain JUnit 5; asserts prompt content against controlled metric inputs
- `src/test/java/com/liatrio/dora/service/InsightsServiceTest.java` — `@ExtendWith(MockitoExtension.class)`; mocks `MetricsService` and `ChatClient`
- `src/test/java/com/liatrio/dora/controller/InsightsControllerTest.java` — `@WebMvcTest(InsightsController.class)`; tests SSE response shape and 503 fallback

### Frontend — Production Code
- `frontend/package.json` — Add `react-markdown` dependency
- `frontend/src/components/InsightsPanel.tsx` — React component: opens native `EventSource`, appends tokens progressively, renders markdown sections, "Regenerate" button, 503 error state, "Generated by Claude" attribution
- `frontend/src/components/Dashboard.tsx` — Add `<InsightsPanel>` below the metric grid when `data` is available
- `docker-compose.yml` — Add `ANTHROPIC_API_KEY` environment variable to the `backend` service

### Frontend — Test Code
- `frontend/src/components/InsightsPanel.test.tsx` — Vitest + React Testing Library; mocks `EventSource` globally

### Notes

- All backend tests run with `./mvnw test`. Run a single test class with `./mvnw test -Dtest=ClassName`.
- All frontend tests run with `cd frontend && npm test`.
- Follow TDD strictly: write the failing test first (RED), implement the minimum to pass (GREEN), then clean up (REFACTOR).
- Use `@ExtendWith(MockitoExtension.class)` for service tests; `@WebMvcTest` for controller tests; plain JUnit 5 for utility class tests.
- Use constructor injection everywhere — no `@Autowired` on fields.
- Log with `private static final Logger log = LoggerFactory.getLogger(ClassName.class);` — never `System.out.println`.
- `ANTHROPIC_API_KEY` must **never** appear in any source file, log output, or committed `.env` file. Use `${ANTHROPIC_API_KEY:}` in `application.properties` so the app starts without the key (with graceful degradation).
- The GitHub PAT must **never** be included in the Claude prompt — only metric values and repo name.
